# 기획서: rbac-menu-management

## 1. 배경 및 목적

- 현재 권한 체계가 `UserRole` enum(super_admin/admin) 2값으로 고정되어 있어 역할 추가·메뉴별 세분화가 불가능하다.
- 사이드바 메뉴가 정적 파일에 하드코딩되어, 메뉴 추가·순서 변경·비활성화 시 코드 배포가 필요하다.
- **해결하려는 문제**: 역할(Role)·메뉴(Menu)·권한(Permission)을 DB 기반으로 관리하여, 코드 변경 없이 접근 제어와 메뉴 구성을 운영할 수 있게 한다.
- **성공 지표**: super_admin이 UI만으로 메뉴 추가/역할 생성/권한 설정을 완료하고, 해당 역할의 관리자가 로그인 시 허용된 메뉴만 사이드바에 표시된다.

## 2. 사용자

- **super_admin**: 메뉴 트리·역할·권한을 생성/수정/삭제하고, 관리자에게 역할을 할당한다.
- **일반 admin**: 자신에게 부여된 역할의 메뉴만 보고, 허용된 작업(읽기/쓰기/삭제)만 수행한다.

## 3. 핵심 기능

### 3-1. 메뉴 관리 (super_admin 전용)
- 메뉴 트리 조회: 2depth 이상 계층 구조를 트리 형태로 표시
- 메뉴 CRUD: 이름·경로·아이콘·상위 메뉴 지정하여 생성/수정/삭제
- 순서 변경: 같은 레벨 내에서 위/아래 버튼으로 순서 조정
- 활성/비활성 토글: 비활성 메뉴는 모든 사용자 사이드바에서 숨김
- 삭제 보호: 하위 메뉴가 존재하면 삭제 차단 (안내 메시지 표시)

### 3-2. 역할 관리 (super_admin 전용)
- 역할 목록 조회 및 CRUD
- SUPER_ADMIN 역할은 시스템 보호 역할로 삭제 차단 (수정은 허용)
- 역할 상세 화면에서:
  - (a) 해당 역할에 관리자 추가/제거
  - (b) 메뉴별 권한 설정: canRead / canWrite / canDelete 체크박스 매트릭스
- 메뉴별 권한은 독립 관리: 상위 메뉴 권한 변경이 하위 메뉴에 자동 전파되지 않음
- 다중 역할 권한 병합: 관리자가 여러 역할을 보유한 경우 각 메뉴·권한의 합집합(OR)을 적용

### 3-3. 관리자 관리 확장 (super_admin 전용)
- 기존 승인/거절/복구 기능에 역할 할당 기능 추가
- 역할 할당은 active 상태인 관리자에게만 가능 (승인 완료 후 별도 할당)
- 관리자 1명에 여러 역할 부여 가능 (다대다)
- 마지막 SUPER_ADMIN 사용자에게서 SUPER_ADMIN 역할 제거 차단 (시스템 잠금 방지)
- 기존 `User.role` enum 필드를 `UserRole` 관계 테이블로 교체

### 3-4. 사이드바 동적 로딩
- 정적 menu-config.ts를 `/auth/me` 응답의 메뉴 트리로 교체
- 메뉴 로딩 중 사이드바에 로딩 상태 표시
- 메뉴 로딩 실패 시 재시도 가능한 에러 상태 표시
- 권한 없는 경로 직접 접근 시 적절한 안내 (403 상태)

### 3-5. Seed 데이터
- 초기 메뉴: 대시보드, 관리자 관리, 메뉴 관리, 역할 관리 (현재 정적 메뉴 + 신규 관리 메뉴)
- 초기 역할: SUPER_ADMIN (전체 권한), ADMIN (대시보드 읽기만)
- 기존 super_admin enum 사용자를 SUPER_ADMIN 역할로 마이그레이션

## 4. 사용자 흐름

### 정상 흐름
1. super_admin이 메뉴 관리에서 새 메뉴를 추가하고 순서를 조정한다.
2. 역할 관리에서 역할을 생성하고, 메뉴별 canRead/canWrite/canDelete를 설정한다.
3. 관리자 관리에서 대상 관리자에게 역할을 할당한다.
4. 해당 관리자가 로그인하면 사이드바에 허용된 메뉴만 표시된다.
5. 허용된 메뉴 내에서 권한에 따라 쓰기/삭제 버튼이 표시되거나 숨겨진다.

### 예외 흐름
1. 하위 메뉴가 있는 메뉴 삭제 시도 → 차단 안내 표시.
2. SUPER_ADMIN 역할 삭제 시도 → 차단 안내 표시.
3. 마지막 SUPER_ADMIN 사용자에게서 역할 제거 시도 → 차단 안내 표시.
4. active가 아닌 관리자에게 역할 할당 시도 → 차단 안내 표시.
5. 역할이 없는 관리자가 로그인 → 빈 사이드바 + "역할이 할당되지 않았습니다" 안내.
6. 권한 없는 경로 직접 접근 → 403 안내 페이지.
7. `/auth/me` 응답 실패 → 사이드바 로딩 에러 + 재시도 버튼.

## 5. 완료 기준

- [ ] `/admin/menus` 화면에서 메뉴 트리 조회·생성·수정·삭제·순서 변경·활성 토글이 동작한다
- [ ] 하위 메뉴가 있는 메뉴는 삭제할 수 없고 안내가 표시된다
- [ ] `/admin/roles` 화면에서 역할 CRUD가 동작한다
- [ ] 역할 상세에서 관리자 추가/제거와 메뉴별 권한(canRead/canWrite/canDelete) 설정이 동작한다
- [ ] `/admin/users` 화면에서 관리자에게 역할을 할당/변경할 수 있다
- [ ] 기존 enum 기반 role이 관계 테이블로 마이그레이션되어 기존 사용자 데이터가 유지된다
- [ ] 로그인 후 사이드바에 사용자 역할에 허용된 메뉴만 표시된다
- [ ] 메뉴 로딩 중/실패 시 각각 로딩 상태와 에러+재시도가 표시된다
- [ ] 권한 없는 경로 직접 접근 시 403 안내가 표시된다
- [ ] Seed 실행 시 초기 메뉴·역할·권한 데이터가 생성된다
- [ ] 메뉴 관리·역할 관리 화면은 super_admin 역할만 접근 가능하다
- [ ] 다중 역할 보유 관리자의 권한이 합집합(OR)으로 적용된다
- [ ] SUPER_ADMIN 역할 삭제가 차단된다
- [ ] 마지막 SUPER_ADMIN 사용자에게서 역할 제거가 차단된다
- [ ] active 상태가 아닌 관리자에게 역할 할당이 차단된다

## 6. 미결 사항

- 역할 삭제 시 해당 역할이 할당된 관리자 처리 정책: (a) 삭제 차단 (b) 역할만 해제 후 삭제 — 결정 필요 (권장: 할당된 관리자가 있으면 삭제 차단)
- 메뉴 트리 depth 상한: 2depth면 충분한지, 3depth 이상도 허용할지 — 결정 필요
