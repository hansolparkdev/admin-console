# 기획서: google-oidc-login

## 1. 배경 및 목적
- 콘솔 접근 통제를 위해 조직 Google 계정 기반 로그인이 필요.
- 개인 Gmail 계정·비인가 이메일은 진입 차단.
- 성공 지표: 인가 사용자만 `(app)` 진입 성공률 100%, 비인가 거부 시 안내 노출 100%.

## 2. 사용자
- 운영 콘솔 접근 권한을 가진 조직 구성원.
- 사용 맥락: 최초 접속, 세션 만료 후 재로그인, 로그아웃 후 재진입.
- 목표: 별도 계정·암호 없이 조직 Google 계정으로 즉시 콘솔 진입.

## 3. 핵심 기능
- Google 계정으로 로그인 시작.
- 로그인 성공 시 콘솔 진입, 실패 시 사유 표시.
- 비인가 계정(개인 Gmail 등) 차단 및 안내.
- 세션 유지 및 로그아웃.
- 미인증 상태에서 보호 경로 접근 시 로그인 화면으로 유도 후 원래 경로 복귀.

## 4. 사용자 흐름
**정상 흐름:**
1. 미인증 사용자가 보호 경로에 접근하거나 로그인 화면을 연다.
2. "Google 계정으로 계속하기"를 누른다.
3. Google 동의 화면을 거쳐 조직 계정으로 인증한다.
4. 콘솔 기본 화면으로 진입한다(원래 요청 경로가 있으면 그곳으로).

**예외 흐름:**
- 개인 Gmail 등 비인가 계정 → 로그인 화면으로 복귀, "조직 계정만 허용" 사유 안내.
- Google 인증 취소/실패 → 로그인 화면 유지, 재시도 가능한 에러 안내.
- 네트워크·서버 오류 → 재시도 가능한 일시 오류 안내.
- 세션 만료 상태에서 보호 경로 접근 → 로그인 화면으로 이동, 로그인 후 원래 경로 복귀.

## 5. 화면 구성
**참조 소스**: `docs/design/login/` 열람 — `screen.png`, `screen.html`, `design-system.md` (3파일 전부). **충실도**: `strict`.

### 화면: 로그인
- **레이아웃**: 좌 55% 다크 패널 + 우 45% 라이트 패널의 스플릿(데스크톱 기준). 좁은 화면에서는 좌 패널 숨김, 우 패널 전폭.
- **좌 패널 구성(위→아래)**:
  - 상단: 워드마크 "ADMIN CONSOLE"(대문자, 넓은 자간) + 보조 라벨 "운영 콘솔".
  - 중앙: 여백(원본에 feature chip 자리 비어 있음 — 비워둔 상태 유지).
  - 하단: "© 2025 Admin Console" 저작권.
  - 배경: 다크 네이비 + 우상단 블루 방사형 글로우.
- **우 패널 구성(카드 내부, 위→아래)**:
  - 이버로우("로그인", primary 색, 대문자·넓은 자간).
  - 헤드라인 "콘솔에 접속하세요".
  - 서브카피 "조직 Google 계정으로 로그인해 주세요.".
  - Google 로그인 버튼(블루 그라디언트, 흰 원 안 Google G 로고 + "Google 계정으로 계속하기").
  - 인포 박스(연회색 배경, info 아이콘 + "조직 계정만 허용됩니다. 개인 Gmail 계정은 거부됩니다.").
  - 구분선 위 법적 고지 "계속 진행하면 이용 약관에 동의하는 것으로 간주됩니다." (이용 약관 링크 포함).
  - 최하단: 좌측 "상태 · 도움말" 링크, 우측 버전 배지 "v1.0"(monospace).
- **상태**: 기본 / 로그인 진행 중(버튼 비활성·로딩 표시) / 로그인 실패(인포 박스 자리 혹은 상단에 사유 메시지).
- **인터랙션**: 버튼 hover 시 그라디언트 심화. 이용 약관·상태·도움말 링크는 hover에서 색 전환.

## 6. API 요구사항
- 로그인 시작: Google OIDC 인가 플로우 개시.
- 콜백 처리: Google 코드 교환 → 조직 도메인/허용 목록 검증 → 세션 발급.
- 세션 조회: 현재 사용자·권한 반환(미인증이면 명확한 미인증 응답).
- 로그아웃: 세션 파기.
- 인증 필요 API는 인증 실패 시 명확한 상태로 응답해 프런트가 로그인 화면으로 유도할 수 있어야 함.
- 모든 인증 판정의 최종 근거는 백엔드. 프런트 표시는 UX 용도.

## 7. 패키지 분담
- **apps/admin**: 로그인 화면, Google 로그인 시작·콜백 수신, 세션 쿠키 기반 상태 표시, 보호 경로 진입 전 인증 확인, 원래 경로 복귀.
- **apps/api**: Google OIDC 토큰 검증, 조직 계정 허용 규칙 집행, 세션 발급·파기, 인증/인가 응답.

### 규약 예외 요청
- 없음. `(public)` 그룹에 로그인 화면 배치, `(app)` 그룹은 인증 필요 Shell이라는 기존 규약을 그대로 따름.

## 9. 제약 조건
- 프런트는 동일 출처 `/api/*`만 호출. 백엔드 URL은 브라우저에 노출하지 않음.
- 세션은 `httpOnly` 쿠키. 토큰을 브라우저 저장소에 보관하지 않음.
- 허용 계정 규칙(도메인·허용 목록)은 서버 설정으로 주입.
- `docs/legacy/**` 불변.

## 10. 완료 기준
- [ ] 미인증 사용자가 보호 경로에 접근하면 로그인 화면으로 이동한다.
- [ ] 로그인 화면이 원본 디자인(스플릿 레이아웃, 좌 패널 워드마크·저작권, 우 패널 이버로우·헤드라인·서브카피·Google 버튼·인포 박스·약관·푸터 링크·버전 배지) 그대로 표시된다.
- [ ] "Google 계정으로 계속하기"를 누르면 Google 동의 화면으로 이동한다.
- [ ] 조직 계정으로 로그인하면 원래 요청 경로 또는 콘솔 기본 화면으로 진입한다.
- [ ] 개인 Gmail 등 비인가 계정은 로그인 화면으로 돌아오고 "조직 계정만 허용" 사유가 표시된다.
- [ ] Google 인증 취소·실패·일시 오류 시 재시도 가능한 안내가 표시된다.
- [ ] 로그인 상태에서 로그아웃하면 세션이 파기되고 이후 보호 경로 접근 시 다시 로그인 화면으로 이동한다.
- [ ] 세션 만료 후 보호 경로 접근 시 로그인 화면으로 이동하고, 로그인 후 원래 경로로 복귀한다.

## 11. 미결 사항
- 허용 조직 도메인·예외 허용 목록의 구체 값(운영 설정 시 확정).
- 세션 유효 기간·갱신 정책(후속 보안 정책 정의 시 확정).
- "상태", "도움말" 링크의 목적지(후속 슬라이스에서 정의).
- 이용 약관 문서 실제 경로.
- 버전 배지 값의 주입 경로(빌드 메타데이터 연동 여부).
